# Example Projects

## feature-team-billing

The canonical end-to-end demo lives in [`examples/feature-team-billing/`](../examples/feature-team-billing/).

It shows the full artifact chain:

1. `spec.md` — product intent, acceptance criteria, and scope boundaries
2. `plan.md` — architecture and implementation steps
3. `tasks.md` — actionable work breakdown
4. `review.md` — completion checklist for spec alignment

Use it as a reference when filling in your own generated specs and plans.
